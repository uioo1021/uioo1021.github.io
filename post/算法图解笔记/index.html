第一章

二分查找

  • 时间复杂度：O(log n)

  • 利用二分查找序列的条件：序列有序

  • 代码

def binary_search(list, item):
    low = 0
    high = len(list) - 1  # 用 -1 而不是 —1
    while low <= high:
        mid = (low + high) // 2  # 确保是整数除法
        guess = list[mid]
        if guess == item:
            return mid
        if guess > item:
            high = mid - 1
        else:
            low = mid + 1
    return None

第二章

数组

  • 优点

    1. 访问速度快：数组支持随机访问（通过索引直接访问元素），其时间复杂度为 O(1)。
    2. 内存连续：数组在内存中是连续存储的，访问时速度较快。
    3. 空间开销小：数组只需要存储数据本身，不需要额外的指针或结构，因此相对较节省空间。

  • 缺点

  1. 固定大小：数组的大小在初始化时就确定，无法动态扩展。如果需要改变数组的大小，需要重新分配一个更大的数组，并将原数据复制到新数组中，这可能非常耗时（O(n)）。
  2. 插入和删除性能差：在数组中插入或删除元素时，需要移动数组中的元素（O(n)），尤其是当操作发生在数组的开始或中间时。
  3. 内存碎片问题：如果数组大小不合适（过大或过小），可能会浪费内存或导致内存不足。

链表

  • 优点
  1. 动态大小：链表的大小可以动态增长或缩小，无需提前知道元素的个数，避免了数组中大小固定的限制。
  2. 插入和删除高效：在链表中插入或删除元素时，只需要修改相邻节点的指针，时间复杂度是 O(1)，这比数组更高效，尤其是在中间或开头插入和删除时。
  3. 内存利用率高：由于链表是按需分配内存的，所以可以灵活使用内存，避免了数组在大小不合适时的浪费。
  • 缺点
  1. 访问速度慢：链表不支持随机访问，查找某个元素时需要从头节点开始按顺序遍历，时间复杂度为 O(n)，因此在查询操作上比数组慢。
  2. 额外的内存开销：链表中的每个节点都需要额外的存储空间来存储指向下一个节点的指针或引用，导致内存消耗比数组大。

选择排序

  • 时间复杂度：O(n^2)

  • 代码

def findSmallest(arr): //找最小的元素索引
    smallest = arr[0]
    smallest_index = 0
    for i in range(1, len(arr)):
        if arr[i] < smallest:
            smallest = arr[i]
            smallest_index = i
    return smallest_index

def selectionSort(arr): //从小到大排序
    newArr = []
    while arr:
        smallest_index = findSmallest(arr)
        newArr.append(arr.pop(smallest_index))
    return newArr

第三章

递归

  • 递归和循环的区别
  1. 递归：递归是指一个函数在定义中直接或间接地调用自身。递归将问题分解成更小的子问题，直到达到最简单的情况（基本情况）来结束递归。
  2. 循环：循环是指通过反复执行一段代码直到满足某个条件为止。
  • 递归的基线条件和递归条件
  1. 基线条件：基线条件是递归函数停止调用自身的条件，它告诉程序什么时候不再继续递归，而是返回最终结果。基线条件非常重要，因为如果没有正确的基线条件，递归会进入无限循环，导致栈溢出错误。
  2. 递归条件：递归条件是递归函数继续调用自身的部分。在递归条件中，问题被分解成一个或多个规模更小的子问题，递归函数会通过调用自身来解决这些子问题。

栈

  • 栈：先进后出

  • 调用栈：每当程序执行一个函数调用时，程序会将当前函数的执行状态（包括局部变量、返回地址等）保存在调用栈中。这使得程序能够在函数调用结束后，恢复到原来的状态，继续执行。

  • 递归调用栈：递归函数也使用调用栈

第四章

分治

  • 基本步骤
  1. 分解：将原问题分解成多个规模较小的子问题，通常是通过递归的方式来进行分解。
  2. 求解：递归地解决这些子问题。如果子问题足够小，可以直接求解（通常称为基本情况或边界条件）。
  3. 合并：将子问题的解合并成原问题的解。

快速排序

  • 基本步骤：
  1. 选择基准值：随机。
  2. 将数组分成两个子数组：小于基准值的元素和大于基准值的元素。
  3. 对这两个子数组进行快速排序。

  • 时间复杂度：O(n*log n) （平均运行时间）

  • 代码

def quicksort(array):
    if len(array) < 2:
        return array
    else:
        pivot = array[0]
        less = [i for i in array[1:] if i <= pivot]
        greater = [i for i in array[1:] if i > pivot]
        return quicksort(less) + [pivot] + quicksort(greater)

大O表示法

  • 常数
  1. 大O表示法中的常量有时候事关重大，这就是快速排序比合并排序快的原因所在。
  2. 比较简单查找和二分查找时，常量几乎无关紧要，因为列表很长时，O(log n)的速度比O(n)快得多。

第五章

散列表

  • 时间复杂度：O(1)

  • 作用：

  1. 模拟映射关系；
  2. 防止重复；
  3. 缓存/记住数据，以免服务器再通过处理来生成它们。
  • 冲突（两个键对应一个值）
  1. 填装因子越低，发生冲突的可能性越小，散列表的性能越高。（一旦填装因子大于0.7，就调整散列表的长度）
  2. 良好的散列函数让数组中的值呈均匀分布；糟糕的散列函数让值扎堆，导致大量的冲突。

第六章

广度优先搜索

  • 时间复杂度：O(V + E)，其中V为顶点数，E为边数。

  • 适用方向：

  1. 第一类问题：从节点A出发，有前往节点B的路径吗？
  2. 第二类问题：从节点A出发，前往节点B的哪条路径最短
  • 有向图和无向图
  1. 有向图：有向图中的边为箭头，箭头的方向指定了关系的方向。
  2. 无向图： 无向图中的边不带箭头，其中的关系是双向的

  • 队列：先进先出

  • bfs代码

from collections import deque

def bfs(graph, start):
    visited = set()
    queue = deque([start])
    visited.add(start)
    
    while queue:
        node = queue.popleft()
        print(node, end=" ")
        
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)

graph = {
    'A': ['B', 'C'],
    'B': ['A', 'D', 'E'],
    'C': ['A', 'F'],
    'D': ['B'],
    'E': ['B', 'F'],
    'F': ['C', 'E']
}

bfs(graph, 'A')

第七章

狄克斯特拉算法

  • 基本步骤
  1. 找出“最便宜”的节点，即可在最短时间内到达的节点。
  2. 对于该节点的邻居，检查是否有前往它们的更短路径，如果有，就更新其开销。
  3. 重复这个过程，直到对图中的每个节点都这样做了。
  4. 计算最终路径。
  • 使用条件
  1. 狄克斯特拉算法只适用于有向无环图
  2. 如果有负权边，就不能使用狄克斯特拉算法

  • 代码

    import heapq

def dijkstra(graph, start):
    distances = {node: float('inf') for node in graph}
    distances[start] = 0
    priority_queue = [(0, start)]

    while priority_queue:
        current_distance, current_node = heapq.heappop(priority_queue)
        if current_distance > distances[current_node]:
            continue

        for neighbor, weight in graph[current_node]:
            distance = current_distance + weight
            if distance < distances[neighbor]:
                distances[neighbor] = distance
                heapq.heappush(priority_queue, (distance, neighbor))

    return distances

graph = {
    'A': [('B', 1), ('C', 4)],
    'B': [('A', 1), ('C', 2), ('D', 5)],
    'C': [('A', 4), ('B', 2), ('D', 1)],
    'D': [('B', 5), ('C', 1)]
}

start_node = 'A'
distances = dijkstra(graph, start_node)

for node, distance in distances.items():
    print(f"{start_node} 到 {node} 的最短距离是 {distance}")

  • 狄克斯特拉算法和广度优先搜索的区别

  1. 狄克斯特拉算法适用于在加权图中查找最短路径。
  2. 广度优先搜索用于在非加权图中查找最短路径。

第八章

贪婪算法

  • 具体思想：每步都选择局部最优解，最终得到的就是全局最优解。

  • 背包问题：贪婪策略不能获得最优解

NP完全问题

  • 定义：不可能编写出可快速解决这些问题的算法，你需要计算所有的解，并从中选出最小/最短 的那个。

  • 解决方法：面临NP完全问题时，最佳的做法是使用近似算法。而贪婪算法易于实现、运行速度快，是不错的近似算法。

  • 旅行商问题

  1. 有一位旅行商。他需要前往5个城市。这位旅行商要前往这n个城市，同时要确保旅程最短。为此，可考虑前往这些城市的各种可能顺序。对于每种顺序，他都计算总旅程，再挑选出旅程最短的路线。因此运行时间为O(n!)

  • 集合覆盖问题
  1. 假设你办了个广播节目，要让全美50个州的听众都收听得到。为此，你需要决定在哪些广播台播出。在每个广播台播出都需要支付费用，因此你力图在尽可能少的广播台播出。广播台名单已给出。每个广播台都覆盖特定的区域，不同广播台的覆盖区域可能重叠。如何找出覆盖全美50个州的最小广播台集合呢？ 列出每个可能的广播台集合，可能的子集有2^n个。在这些集合中，选出覆盖全美50个州的最小集合。
  2. 近似算法：

  • 识别NP完全问题
  1. 元素较少时算法的运行速度非常快，但随着元素数量的增加，速度会变得非常慢。
  2. 涉及“所有组合”的问题通常是NP完全问题。
  3. 不能将问题分成小问题，必须考虑各种可能的情况。这可能是NP完全问题。
  4. 如果问题涉及序列（如旅行商问题中的城市序列）且难以解决，它可能就是NP完全问题。
  5. 如果问题涉及集合（如广播台集合）且难以解决，它可能就是NP完全问题。
  6. 如果问题可转换为集合覆盖问题或旅行商问题，那它肯定是NP完全问题。

第九章

动态规划

  • 使用条件
  1. 使用动态规划时，要么考虑拿走整件商品，要么考虑不拿，而没法判断该不该拿走商品的一部分。但使用贪婪算法可轻松地处理这种情况！首先，尽可能多地拿价值最高的商品；如果拿光了，再尽可能多地拿价值次高的商品，以此类推。
  2. 仅当每个子问题都是离散的，即不依赖于其他子问题时，动态规划才管用

最长公共子串

  • 具体步骤

对于最长公共子串问题，答案为网格中最大的数字，它可能并不位于最后的单元格中。

  • 代码
def longest_common_substring_length(word_a, word_b):
    m, n = len(word_a), len(word_b)
    cell = [[0] * (n + 1) for _ in range(m + 1)]
    
    longest_length = 0
    
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if word_a[i - 1] == word_b[j - 1]:
                cell[i][j] = cell[i - 1][j - 1] + 1
                longest_length = max(longest_length, cell[i][j])
            else:
                cell[i][j] = 0

    return longest_length

word_a = "abcdefg"
word_b = "zcdemfgh"
result = longest_common_substring_length(word_a, word_b)
print(result)

    def longestCommonSubsequence(word_a, word_b):
    m = len(word_a)
    n = len(word_b)
    cell = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if word_a[i - 1] == word_b[j - 1]:
                cell[i][j] = cell[i - 1][j - 1] + 1
            else:
                cell[i][j] = max(cell[i - 1][j], cell[i][j - 1])

    return cell[m][n]

第十章

K最近邻算法

  • 邻居的定义：给定一个待预测的数据点，KNN算法通过计算该点与训练数据中所有其他点的距离来选择K个最相似的邻居。

  • K的选择：

  1. K值：即“最近邻”的数量。常常需要根据数据的特点来选择K值。
  2. 如果K值过小，模型容易对噪声数据过拟合；如果K值过大，模型可能过于简单，导致欠拟合。
  • 距离度量：KNN算法的核心是计算数据点之间的距离。最常用的距离度量方法是欧几里得距离

  • 分类：KNN常用于分类问题，分类就是编组。对于待分类的数据点，算法根据其K个最近邻的数据点的类别，采用多数投票原则来确定该数据点的类别。

  • 回归：回归就是预测结果。在回归问题中，KNN预测的是一个数值，而不是类别。它通过K个邻居的平均值或加权平均值来进行预测。

机器学习

  • 光学字符识别（OCR）：OCR的第一步是查看大量的数字图像并提取特征，这被称为训练。OCR中的特征提取要复杂得多，但再复杂的技术也是基于KNN等简单理念的。这些理念也可用于语音识别和人脸识别。

  • 朴素贝叶斯分类器：朴素贝叶斯分类器能计算出邮件为垃圾邮件的概率，其应用领域与KNN相似。

第十一章

二叉查找树

  • 优点：二叉查找树的插入和删除操作的速度快。

  • 缺点：不能随机访问。

反向索引

  • 定义：一个散列表，将单词映射到包含它的页面。

  • 应用：搜索引擎

傅里叶变换

  • 应用：
  1. 音频压缩（如MP3）：傅里叶变换可以将音频信号分解为不同的频率成分，帮助识别哪些音符对整体音频的贡献较大，哪些可以忽略，从而实现数据压缩。例如，在MP3格式中，傅里叶变换使得只保留重要频率成分，删除不重要的部分，降低文件大小。
  2. 图像压缩（如JPG）：JPG图像压缩同样利用傅里叶变换（更常见的是离散余弦变换）将图像信号分解为频率成分，从而实现压缩。只保留重要的频率成分，去除不显著的细节，使得图像文件变小。
  3. 音乐识别（如Shazam）：傅里叶变换可以将音乐信号分解为不同的频率成分，这些频率特征可以用于比较和识别歌曲。例如，Shazam利用傅里叶变换来识别你所听到的音乐，通过匹配音频频谱来找到匹配的歌曲。
  4. 地震预测和DNA分析：傅里叶变换也被用于地震波信号的分析，帮助科学家识别潜在的地震活动模式；同时，它在基因组学中也有重要应用，分析DNA的频率特征，帮助研究基因表达模式等。

并行算法

  • 定义：笔记本电脑和台式机转而采用多核处理器。为提高算法的速度，你需要让它们 能够在多个内核中并行地执行

  • 提升速度的难点：

  1. 并行性管理开销：合并也需要时间。
  2. 负载均衡：均匀地分配工作，让各个内核都一样忙

分布式算法

  • 定义：在并行算法只需两到四个内核时，完全可以在笔记本电脑上运行它，但如果需要数百个内核呢？在这种情况下，可让算法在多台计算机上运行。分布式算法非常适合用于在短时间内完成海量工作。

  • MapReduce（一种流行的分布式算法）

  1. 映射函数：接受一个数组，并对其中的每个元素执行同样的处理。映射是将一个数组转换为另一个数组。

  2. 归并函数：归并是将一个数组转换为一个元素。

布隆过滤器

  • 定义：布隆过滤器是一种概率型数据结构，它提供的答案有可能不对，但很可能是正确的。为判断网页以前是否已搜集，可不使用散列表，而使用布隆过滤器。使用散列表时，答案绝对可靠，而使用布隆过滤器时，答案却是很可能是正确的。

  • 优点：占用的存储空间很少。使用散列表时，必须存储Google搜集过的所有URL，但使用布隆过滤器时不用这样做。

HyperLogLog

  • 定义：近似地计算集合中不同的元素数。它不能给出准确的答案，但也八九不离十，而占用的内存空间却少得多。

  • 面临海量数据且只要求答案八九不离十时，可考虑使用概率型算法。

安全散列算法

  • 定义：给定一个字符串，SHA返回其散列值。对于每个不同的字符串，SHA生成的散列值都不同。

  • 应用：

  1. 使用SHA来判断两个文件是否相同：计算文件的SHA散列值，再对结果进行比较。
  2. 检查密码：在不知道原始字符串的情况下对其进行比较。保证了密码的安全性。

Simhash

  • 定义：SHA还有一个重要特征，那就是局部不敏感的。修改其中的一个字符，再计算其散列值，结果将截然不同。有时候我们希望散列函数是局部敏感的。在这种情况下，可使用Simhash。如果你对字符串做细微的修改，Simhash生成的散列值也只存在细微的差别。这让你能够通过比较散列值来判断两个字符串的相似程度。

  • 应用：需要检查两项内容的相似程度时，Simhash很有用。

Diffie-Hellman 密钥交换

  • 优点
  1. 双方无需知道加密算法。他们不必会面协商要使用的加密算法。
  2. 要破解加密的消息非常困难
